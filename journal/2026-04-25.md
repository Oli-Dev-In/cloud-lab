## Learned
- VSCode: professional code editor with integrated terminal, Git support, syntax highlighting, error detection, and extensions — purpose-built for professional development work
- Git: version control system tracking every file change, allowing return to any previous state and team collaboration
- GitHub: public platform hosting Git repositories — Git works locally, GitHub makes it visible and accessible
- Git three-zone workflow: Working Directory → Staging Area → Local Repository → GitHub
- Git commands: `git add` moves files to staging, `git commit` creates a local snapshot, `git push` transfers commits to GitHub
- Virtualization: allows a physical machine to allocate resources to virtual machines — solves hardware waste, enables isolated and disposable environments
- Hypervisor Type 1 (bare metal) vs Type 2 (hosted): Type 1 runs directly on hardware (AWS Nitro, VMware ESXi), Type 2 runs on top of an OS (VirtualBox)
- ISO disk image: bit-for-bit copy of an installation disc — virtual equivalent of a physical DVD, used to install OS on VMs and conceptually similar to AWS AMIs
- Ubuntu Server: no GUI, terminal only — standard in cloud production environments, most widely used on AWS EC2

## Introduced — to be studied in depth later
- Linux login prompt: `user@hostname:~$` — tilde means home directory, $ means normal user vs # for root
- SSH: protocol for secure remote connection to Linux servers — fundamental for cloud engineering, will be configured tomorrow
- LVM (Logical Volume Manager): allows partition resizing without reformatting — standard in production Linux environments
- VirtualBox snapshots: capture complete VM state at a point in time — allows restoration if something breaks
